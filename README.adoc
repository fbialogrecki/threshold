= Threshold
:toc:
:toclevels: 2

Threshold is an open-source social and event-discovery platform for local music scenes. It combines chronological community feeds, public profiles and pages, groups, events, media, notifications, and organizer-managed guestlists without algorithmic ranking or third-party tracking.

NOTE: Active MVP development. APIs, data models, and deployment procedures may change before the first stable release.

== Product direction

Threshold focuses on direct community relationships:

* chronological content from followed people, pages, groups, and events;
* public artist, venue, collective, promoter, and user profiles;
* groups, posts, comments, votes, reactions, and moderation;
* public event discovery, follows, boosts, lineups, and updates;
* private organizer/DJ-managed guestlists with quotas and QR check-in;
* privacy-first operation without third-party trackers.

The MVP excludes ticketing, payments, direct messages, creator monetization, mobile applications, and algorithmic feeds.

== Architecture

[cols="1,3",options="header"]
|===
|Component |Responsibility
|`apps/web` |Next.js App Router frontend and browser-facing BFF
|`services/users` |Product authentication, sessions, profiles, pages, follows, and canonical blocks
|`services/social` |Groups, posts, comments, reactions, chronological feed, and moderation
|`services/events` |Event CRUD, public event reads, follows, boosts, guestlists, and check-in
|`services/media` |Image validation, metadata, derivatives, and S3-compatible storage
|`services/auth-gateway` |Authentik-backed SSO edge for internal and administrative surfaces
|`libs/py` |Shared Python helpers
|`libs/proto` |Internal Protobuf contracts
|===

Runtime stack:

* Web: Next.js, React, TypeScript, Tailwind CSS, Bun
* Services: Python 3.13, FastAPI, SQLAlchemy, Alembic
* Data: PostgreSQL via CloudNativePG; S3-compatible object storage
* Messaging: NATS Core
* Secrets: OpenBao and External Secrets Operator
* Delivery: Woodpecker CI, Harbor, Argo CD, Kustomize, Kubernetes
* Observability: OpenTelemetry and the Grafana LGTM stack

== Getting started

[source,bash]
----
git clone https://github.com/fbialogrecki/threshold.git
cd threshold
uv sync --frozen
go-task web:install
cp apps/web/.env.example apps/web/.env.local
go-task web:dev
----

Open http://localhost:3000[]. The example environment file contains development placeholders only. Never commit `.env.local`, credentials, tokens, kubeconfigs, or rendered Kubernetes secrets.

The full integrated environment requires PostgreSQL, NATS, object storage, and private deployment configuration.

== Contributing

Focused issues and pull requests are welcome. Keep changes inside the stated MVP scope, add tests for changed behavior, run the quality gates, and never commit secrets or private operational data.

Untrusted pull-request jobs run without release, registry-push, signing, Kubernetes, or GitOps credentials.

The manual trusted release pipeline builds from the exact public commit, pushes immutable image digests, and opens a digest-only change in the private GitOps repository. Product source is never synchronized through a private mirror. Release credentials and environment-specific values are configured only in the trusted Woodpecker installation.
